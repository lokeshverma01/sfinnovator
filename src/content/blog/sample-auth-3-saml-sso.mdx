## How SAML SSO works

The IdP asserts the user's identity to Salesforce (the service provider) via a signed
SAML assertion. No password is stored in Salesforce.

## Configure Salesforce

1. Setup → Single Sign-On Settings → enable SAML.
2. Add the IdP metadata (entity ID, login URL, certificate).

## Test it

Use the IdP-initiated login and confirm the user lands in Salesforce.
