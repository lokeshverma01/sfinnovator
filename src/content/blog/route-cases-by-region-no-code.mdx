## The situation

The team was manually triaging every incoming case and assigning it by hand. This was
slow, error-prone, and didn't balance load across agents.

## Why not Apex?

Apex routing would work, but it's harder to change, needs a developer for every tweak,
and adds test-coverage overhead. The declarative stack handles this scenario cleanly and
stays in the admin's hands.

## The approach

1. **Create three queues** — `Support AMER`, `Support EMEA`, `Support APAC` — each with
   the relevant agents as members.
2. **Case Assignment Rules** — one rule entry per region, matching on the case's
   `Region` field:

   ```
   Case: Region equals AMER  →  assign to "Support AMER" queue
   Case: Region equals EMEA  →  assign to "Support EMEA" queue
   Case: Region equals APAC  →  assign to "Support APAC" queue
   ```

3. **Omni-Channel** — enable it, create a routing configuration with a capacity model,
   and attach each queue so cases flow to the next available agent.

## The result

Cases now land in the right regional queue the instant they're created, and Omni-Channel
balances them across agents by capacity. The whole flow is configurable by an admin with
no code to maintain.
