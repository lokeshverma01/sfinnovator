## What this flow is

The web server flow is for server-side apps that can protect a client secret. The user
authorizes once; your server swaps the returned code for tokens.

## Set up the Connected App

1. Setup → App Manager → New Connected App.
2. Enable OAuth, set the callback URL, pick scopes.

```bash
# Exchange the code for tokens
curl https://login.salesforce.com/services/oauth2/token \
  -d grant_type=authorization_code -d code=$CODE \
  -d client_id=$CID -d client_secret=$SECRET -d redirect_uri=$CB
```

## Test it

Walk the redirect, capture the code, exchange it, and confirm you get an access token.
