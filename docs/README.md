# SF Innovator — Documentation

Living documentation for the project. Updated **in the same commit** as the code it
describes. Start here:

| Document                                             | What it's for                                                        |
| ---------------------------------------------------- | -------------------------------------------------------------------- |
| **[AUTHORING.md](./AUTHORING.md)**                   | **No-code publishing: add a post/solution by filling one file.**     |
| [DEV_SETUP.md](./DEV_SETUP.md)                       | Connect VS Code to GitHub, edit locally, preview, push to deploy.    |
| [templates/](./templates/)                           | Copy-paste starter files for each post type + portfolio solution.    |
| [IMPLEMENTATION_GUIDE.md](./IMPLEMENTATION_GUIDE.md) | Step-by-step build journal. Follow it to rebuild the site from zero. |
| [COMPONENT_REFERENCE.md](./COMPONENT_REFERENCE.md)   | Map of every file/component: path, purpose, props, dependencies.     |
| [TROUBLESHOOTING.md](./TROUBLESHOOTING.md)           | "If X fails, look here" — pinpoint the file, fix, and confirm.       |
| [SECURITY.md](./SECURITY.md)                         | Security posture, controls, verification, and go-live checklist.     |
| [CONVENTIONS.md](./CONVENTIONS.md)                   | Coding & component standards (reusability, comments, a11y, git).     |

See also [`../PROJECT.md`](../PROJECT.md) for the high-level roadmap and decision log.

## The documentation rule

> Every phase/feature updates these docs alongside the code. A change isn't "done" until
> its documentation is too (see the Definition of Done in `CONVENTIONS.md`).
